It is designed to be clear for both developers and project stakeholders.📧 Automated Email-to-API Sales Order ProcessorAn efficient n8n workflow that automates the extraction of sales data from email attachments. It listens for incoming emails, parses XML attachments, and pushes the structured data to a downstream API endpoint in real-time.🚀 OverviewManual data entry from email orders is slow and error-prone. This workflow eliminates that bottleneck by:Monitoring an IMAP mailbox for new orders.Extracting XML attachments automatically.Converting raw binary data into structured JSON.Forwarding the data to a Sales Order API via POST request.🛠 Workflow ArchitectureThe workflow consists of five primary stages:Trigger (IMAP): Watches for new emails and downloads binary attachments.Data Conversion: Moves the binary buffer of the attachment into a string variable.Parsing: Transforms the XML string into an object that n8n can manipulate.Data Mapping: Uses a Set node to format the data according to the API requirements.Transmission: Sends the final payload to the local sales management system.📦 Prerequisitesn8n (Desktop or Docker version)IMAP Access: An email account with IMAP enabled (e.g., Gmail with an App Password).Sales API: A listener running at http://localhost:5679 (or update to your production URL).⚙️ Setup InstructionsImport the Workflow:Copy the JSON code for the workflow.In n8n, click on the + icon and select Import from JSON.Configure Credentials:Open the IMAP Email node.Add your credentials (Host, Port, User, and Password).Adjust the API Endpoint:Open the HTTP Request node.Update the URL to match your internal or cloud-based API endpoint.Mapping Fields:In the Set node, ensure the orderRequest value correctly maps to the specific tags within your XML structure.📄 Configuration DetailsNodeParameterValueIMAPdownloadAttachmentstrueMove Binaryencodingutf8XMLignoreAttrstrueHTTP RequestbodyContentTypeform-urlencoded🛡 Security NoteThis workflow currently has allowUnauthorizedCerts set to true for local development.
